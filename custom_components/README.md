<p>The <b>custom_components</b> folder is essential for the installation of third-party extensions such as <a href="https://hacs.xyz/" target="_blank">HACS</a> and all the extensions that can be installed from it.</p>

Some of the extensions or add-ons I have installed are:

- <a href="https://hacs.xyz/" target="_blank">HACS</a> (massive store for third-party add-ons)
- <a href="https://github.com/koying/openrgb_ha" target="_blank">OpenRGB</a> (for controlling RGB equipment v√≠a open software)
- <a href="https://github.com/custom-components/authenticated" target="_blank">Authenticated</a> (for controlling which devices are authorized to log-in)
- <a href="https://github.com/nielsfaber/alarmo" target="_blank">Alarmo</a> (easy to set-up alarms for Home Assistant based on sensors)
- <a href="https://github.com/rgruebel/ha_zigbee2mqtt_networkmap" target="_blank">Zigbee2mqtt_networkmap</a> (for visualization of a Zigbee network)
- <a href="https://github.com/kuchel77/diskspace" target="_blank">Diskspace</a> (for controlling space left on main disk)
- <a href="https://github.com/dckiller51/bodymiscale" target="_blank">Bodymiscale</a> (for extrapolating data from a Xiaomi Mi Scale 1/2; also requires the <b>config/components</b> folder to exist)
- <a href="https://github.com/heyajohnny/cryptoinfo" target="_blank">Cryptoinfo</a> (for gathering info about different cryptocurrencies)

<p>Please refer to the creators of each add-on for how-to instructions and don't forget to endorse their work.</p>
